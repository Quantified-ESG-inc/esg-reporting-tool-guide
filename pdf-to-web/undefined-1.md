# 자주 묻는 질문 및 우수 사례

자주 묻는 질문 바로가기





#### 매년 수천만 원씩 드는 디자인 비용, 정말 줄어드나요?

PDF는 매년 새로운 표지와 내지 레이아웃을 잡기 위해 디자인 외주 비용이 반복적으로 발생합니다.

* ERT의 해답: 한 번 구축된 메인 페이지 디자인을 '복제(Clone)' 기능을 통해 내년에도 그대로 활용할 수 있습니다.
* 실무 이득: 전년도 보고서를 복제해 내용만 업데이트하면 되므로, 반복적인 디자인 외주 비용과 의사결정 시간을 획기적으로 줄여 디자인을 '소비'가 아닌 '자산'으로 만들 수 있습니다.

#### 시간은 어떻게 단축되나요?

ERT를 사용하면 보고서 발간의 전체 공정 중 가장 시간이 많이 걸리는 \*\*'디자인'\*\*과 '데이터 검증' 단계를 획기적으로 줄일 수 있습니다.

* AI PDF-to-HTML 자동 변환: 기존 PDF 자료를 일일이 옮길 필요 없이 AI가 웹 구조로 자동 변환하여 초기 세팅 시간을 며칠 내로 단축합니다.
* 실시간 시각화: 디자인 업체에 도표를 의뢰하고 시안을 기다릴 필요가 없습니다. 데이터를 입력하는 즉시 전문적인 인터랙티브 차트가 생성됩니다.
* 협업 가속화: 타 부서 담당자가 직접 시스템에 접속해 데이터를 넣고 코멘트를 다는 과정을 통해 이메일과 전화로 소모되던 시간을 없앱니다.

#### 기존에 만든 PDF 자료들을 일일이 옮겨야 하나요?

가장 먼저 걱정되는 것은 과거 데이터의 이관 작업입니다. 처음부터 다시 타이핑해야 한다면 디지털 전환은 또 다른 '일감'이 되기 때문입니다.

* ERT의 해답: AI 기반의 PDF-to-HTML 자동 변환 기능을 통해 기존 PDF를 업로드하기만 하면 웹 형태로 즉시 재구성됩니다.
* 실무 이득: 처음부터 웹 페이지를 기획하고 텍스트를 복사할 필요 없이, 아주 짧은 시간에 초기 디지털 세팅을 마칠 수 있습니다.

#### 평가 기관이 우리 웹사이트의 도표 내용을 읽을 수 있나요?

기존 웹 보고서들은 도표나 조직도를 '이미지'로 넣어 검색 엔진이나 평가 기관의 크롤링 봇이 내용을 읽지 못하는 경우가 많았습니다.

* ERT의 해답: ERT는 모든 도식과 성과 지표를 HTML 코딩 기반으로 구현하여, 이미지가 아닌 '텍스트' 데이터로 인식되게 합니다.
* 실무 이득: 구글 등 서치 엔진 최적화(SEO)는 물론, 평가 기관이 데이터를 수집할 때 누락 없이 우리 기업의 성과를 읽어갈 수 있어 평가 점수 관리에 매우 유리합니다.

#### 데이터 오류가 나면 어쩌죠? 히스토리 관리는요?

실시간 수정이 가능한 웹의 특성상, 과거의 확정된 기록이 유실되거나 수기 입력 시 오타가 날까 봐 걱정할 수 있습니다.

* 데이터 무결성: 엑셀 데이터를 시스템에 직접 연동하여 정량 데이터를 웹으로 구현하므로 수기 입력 오류를 원천 차단하고 논리 검수까지 진행합니다.
* 이원화 아카이빙: 최신 정보는 인터랙티브 웹으로 보여주되, 각 연도별 확정본은 PDF 형태로 별도 보관하여 완벽한 히스토리 관리를 지원합니다.

#### PDF 보고서도 필요한데 따로 제작해야 하나요?

ERT는 웹 보고서의 데이터를 기반으로 고품질의 PDF를 자동 생성하고 관리하는 시스템을 갖추고 있습니다.

* 동시 아카이빙 시스템: 웹 공시와 동시에 해당 시점의 보고서 내용을 PDF 형태로 자동 저장하여 히스토리를 관리합니다.
* 데이터 일치성 100%: 웹에 입력된 정량·정성 데이터가 그대로 PDF로 변환되므로, 두 매체 간의 수치가 달라지는 오류를 원천 차단합니다.
* 별도 제작 공정 생략: PDF를 위한 레이아웃 수정, 오탈자 교정, 디자인 외주 과정을 거치지 않아도 되므로 발간 리소스가 획기적으로 줄어듭니다.
