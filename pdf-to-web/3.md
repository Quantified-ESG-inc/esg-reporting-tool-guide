### **📌 PDF 보고서를 웹으로** 전환하는 경우

기존에 발간하던 PDF 보고서나 확정된 목차안이 있다면, 첫 번째 옵션인 **\[새 목차 생성하기]를** 선택하는 것이 가장 효율적입니다.

우리 기업 고유의 보고서 흐름과 스토리텔링을 유지하면서, 구조만 웹 사이트맵(Sitemap) 형태로 최적화하여 빠르게 이식할 수 있기 때문입니다. 아래 가이드를 통해 기존 문서를 웹 메뉴로 구조화하는 방법을 확인해 보세요.

{% step %}
### 1단계. 새 목차 생성 시작하기

* 보고서를 선택해 편집 화면으로 이동합니다.
* 목차 작성 방식 선택 화면에서 첫 번째 항목인 \[새 목차 생성]을 클릭합니다.
* PDF 목차를 활용해 웹 보고서의 목차를 작성할 수 있습니다.&#x20;
{% endstep %}

{% step %}
### 2단계. 대분류 및 소분류 구성하기

편집 화면에 진입하면 본격적으로 보고서의 뼈대를 잡을 수 있습니다.

PDF의 가장 큰 챕터(Chapter) 제목들은 웹사이트의 상단 메인 메뉴가 되도록 작성해 주세요.

* 목차 추가하기: 화면 중앙의 \[+ 목차 추가하기] 버튼을 클릭하여 첫 번째 목차 항목을 생성합니다.
* 항목 입력: \* 대분류: 보고서의 큰 장(Chapter) 제목을 입력합니다. (예: Intro, Environmental, Social 등)
* 소분류: 해당 장에 포함될 세부 항목을 입력합니다. (예: 온실가스 배출량 관리, 안전보건 정책 등)
* 소분류 추가: 소분류 입력창 아래의 \[+] 버튼을 누르면 해당 대분류 안에 더 많은 소분류 항목을 계속해서 추가할 수 있습니다.
  * _Web Tip: 소분류는_ 대분류를 클릭했을 때 펼쳐지는 세부 항목입니다. 가독성을 위해 PDF와  달리 웹에서는 내용이 적은 항목은 과감히 통합해야 합니다.
*
{% endstep %}

{% step %}
### 3단계. 목차 확장 및 저장하기

* 대분류 추가: 하나의 대분류 작성이 끝났다면, 하단의 \[+ 목차 추가하기] 버튼을 다시 눌러 다음 대분류를 생성합니다.
* 목차 저장: 모든 목차 구성이 완료되었다면 화면 좌측 하단의 \[목차 저장하기] 버튼을 클릭하여 설정을 완료합니다.
{% endstep %}
{% endstepper %}

{% hint style="info" %}
#### 💡 작성 팁: 웹사이트에 최적화된 메뉴 구성법

**1. 메뉴명은 '키워드' 중심으로 짧게 (Mobile First)**&#x20;

* 웹사이트 상단 공간은 한정적입니다.&#x20;
* "Sustainable Management & Performance" 처럼 긴 제목 대신, "Environmental" 혹은 "ESG Data" 처럼 핵심 키워드로 간결하게 입력하세요.

**2. 대분류는 5\~7개가 적당합니다**&#x20;

* 사용자가 한눈에 파악할 수 있도록 상단 메뉴는 5\~7개로 유지하는 것을 권장합니다.&#x20;
* 기존 PDF 목차가 너무 많다면 유사한 챕터끼리 통합하여 조정해 보세요.

**3. 소분류는 6개를 넘기지 마세요**&#x20;

* 클릭했을 때 펼쳐지는 메뉴가 너무 길면 가독성이 떨어집니다. 내용이 적은 항목은 과감히 통합하여 최대 6개를 넘지 않도록 구성하세요.&#x20;
* 만약 항목이 너무 많다면 메뉴를 늘리는 대신 페이지 내부의 네비게이션 바(Tab) 기능을 활용하는 것이 좋습니다.
{% endhint %}
