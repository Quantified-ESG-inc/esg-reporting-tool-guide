### 팀원 초대 및 내보내기

성공적인 보고서 발간을 위해 각 부서의 데이터 담당자(Data Owner)를 초대하고, 권한을 설정하여 효율적인 협업 환경을 조성합니다.

{% stepper %}
{% step %}
#### &#x20;1단계. 팀 관리 화면 진입하기

* 메뉴 이동: ERT 메인 화면 좌측 하단 또는 상단 프로필 영역의 \[설정] 혹은 \[팀 관리] 메뉴를 클릭합니다.
* 멤버 목록 확인: 현재 대시보드에 참여 중인 팀원 목록과 초대 상태를 한눈에 확인할 수 있습니다.
{% endstep %}

{% step %}
#### 2단계. 팀원 초대하기

* 초대하기 클릭: 화면 우측 상단의 \[+ 팀원 초대하기] 버튼을 클릭합니다.
* 정보 입력: \* 이메일 주소: 초대할 팀원의 회사 이메일 주소를 입력합니다.&#x20;
{% endstep %}

{% step %}
#### 3단계. 초대 수락 및 협업 시작하기

* 초대 메일 발송: \[초대 보내기] 버튼을 누르면 해당 팀원에게 가입 안내 메일이 발송됩니다.
* 참여 확인: 초대받은 팀원이 메일 내 링크를 통해 접속하면 팀원 목록이 '초대 대기'에서 '활성' 상태로 변경됩니다.
* 업무 할당: 이제 초대된 팀원들에게 \[3단계]에서 구성한 목차별로 담당 지표를 배정하고 데이터 수집을 요청할 수 있습니다.
{% endstep %}
{% endstepper %}



***

#### 💡 작성 팁: 효과적인 팀 구축 노하우

* 데이터 소유자(Data Owner) 선점: 환경(환경팀/시설팀), 사회(인사팀/총무팀), 지배구조(법무팀/재무팀) 등 각 목차에 필요한 데이터를 가장 잘 아는 실무자를 초대하세요.
* 최종 검토자 포함: 보고서 발행 전 내부 검증을 위해 팀장님이나 임원분들을 미리 초대하여 실시간으로 진행 상황을 공유하는 것을 추천합니다.

### 팀원 권한 관리하기

ERT에서는 팀원의 역할에 따라 보고서의 수정 범위와 관리 권한을 차등적으로 부여할 수 있습니다. 이 기능은 보안이 중요한 데이터나 특정 부서만 담당해야 하는 항목을 관리할 때 매우 유용합니다. 우리 팀원들이 각각 본인에게 배정된 역할에만 집중할 수 있도록 목차별 접근 권한을 설정해 보세요.

{% stepper %}
{% step %}
#### 1단계. 목차별 권한 설정 메뉴 진입하기

* 항목 선택: 왼쪽 목차 리스트에서 권한을 설정하고 싶은 특정 섹션(대분류 또는 소분류)을 선택합니다.
* 권한 관리 클릭: 해당 항목 우측 상단이나 메뉴에 위치한 \[권한 설정] 혹은 자물쇠 아이콘을 클릭합니다.
{% endstep %}

{% step %}
#### 2단계. 접근 허용 팀원 지정하기

* 팀원 선택: 현재 팀에 가입된 멤버 목록이 나타납니다. 해당 섹션에 접근하여 내용을 작성하거나 수정해야 하는 담당자를 선택합니다.
* 권한 범위 설정: \* 접근 허용: 선택된 팀원만 해당 섹션을 보고 편집할 수 있습니다.
  * 접근 제한: 선택되지 않은 팀원에게는 해당 섹션이 비활성화되거나 보이지 않게 설정됩니다.
{% endstep %}

{% step %}
#### 3단계. 설정 저장 및 적용 확인하기

* 저장하기: 권한 설정이 완료되었다면 \[확인] 또는 \[저장] 버튼을 클릭합니다.
* 상태 확인: 목차 리스트에서 권한 제한이 걸린 항목은 아이콘 표시를 통해 관리자 및 허용된 인원만 접근 가능함을 확인할 수 있습니다.
{% endstep %}
{% endstepper %}





***

#### 💡 작성 팁: 효율적인 권한 관리를 위한 제언

* 부서별 설정: 인사 데이터는 인사팀 담당자에게만, 환경 데이터는 환경팀 담당자에게만 접근 권한을 부여하세요. 협업 과정에서 발생할 수 있는 혼선을 줄이는 가장 좋은 방법입니다.
* 관리자의 역할: '관리자(Admin)' 권한을 가진 유저는 모든 섹션의 권한 설정을 제어할 수 있으므로, 전체적인 협업 흐름을 모니터링하며 필요 시 권한을 유연하게 조정해 주는 것이 좋습니다.



### 협업하기

이렇게 초대된 팀원과는 실시간으로 소통하며 보고서를 함께 작성할 수 있습니다. 각 부서의 담당자들이 모여 하나의 완성된 보고서를 만들어가는 과정을 ERT가 더 쉽고 빠르게 도와드릴게요.

### 팀원과 함께 보고서 작성하기

#### 실시간 댓글로 나누는 피드백

작성 중이거나 검토가 필요한 섹션에 마우스 커서를 올리면 펜 아이콘이 활성화됩니다. 이 아이콘을 클릭하여 팀원들과 자유롭게 의견을 나눠보세요.

* 댓글 남기기: 특정 섹션에 대한 수정 보완 사항이나 의견을 즉시 입력할 수 있습니다.
* 팀원 호출(@Mention): `@` 기호를 입력하고 팀원 이름을 선택해 보세요. 해당 팀원에게 실시간 알림이 전송되어 더욱 빠른 소통이 가능합니다.

#### 투명한 진행 상태 관리

각 섹션 상단의 상태 라벨을 통해 보고서가 어느 정도 완성되었는지 실시간으로 확인할 수 있습니다.

* 작성 중: 담당 부서에서 원고를 채워넣고 있는 단계입니다.
* 검토 요청: 작성이 완료되어 관리자나 최종 검토자의 피드백을 기다리는 상태입니다.
* 완료: 모든 검토가 끝나 최종 확정된 섹션입니다.

#### 버전 관리 및 수정 히스토리

누가, 언제, 어떤 내용을 수정했는지 기록이 남기 때문에 데이터의 정합성을 안전하게 관리할 수 있습니다.

* 이력 추적: 원고가 수정될 때마다 자동으로 버전이 기록되어 이전 내용과 쉽게 대조해 볼 수 있습니다.
* 데이터 보호: 실수로 내용이 변경되더라도 히스토리를 통해 이전 버전을 확인하거나 복구할 수 있어 안심하고 작업할 수 있습니다.

### 소유 권한 양도하기

소유 권한 양도는 워크스페이스의 모든 제어권을 넘기는 중요한 과정입니다. 보안상 시스템에서 직접 변경할 수 없으므로, 아래 안내에 따라 고객센터로 요청해 주세요.

***

#### 소유 권한 양도 안내

> 보안을 위해 직접 변경이 제한됩니다. 원활한 서비스 운영과 데이터 보호를 위해 소유권 변경은 QESG 관리자가 직접 지원해 드리고 있습니다.

진행 과정

* 준비: 권한을 양도받으실 팀원의 정보(성함, 계정 이메일)를 확인합니다.
* 요청: 화면 우측 하단의 \[[문의하기](https://odvvp.channel.io)] 버튼을 클릭하여 소유권 양도를 요청합니다.
* 완료: QESG 관리자가 확인 절차를 거친 후 권한 양도을 완료하고 결과를 안내해 드립니다.

***

💡 꼭 확인해 주세요!

* 소유권을 양도하면 이전 소유자는 더 이상 관리자 기능을 사용할 수 없게 됩니다.
* 신속한 처리를 위해 반드시 워크스페이스에 가입된 팀원 중에서 양도 대상을 선택해 주세요.
